## 📋 週次アップデートのプロンプト（コピーして使う）

```
「週刊 AI実験ノート」の index.html に今週の学びを追加してください。

以下の情報を元に、weeklyData 配列に新しいエントリーオブジェクトを追記してください。
既存のエントリーは消さずに、配列の末尾（または先頭）に追加してください。

--- 今週の情報 ---

Week番号: [例: 2]
日付: [例: 2026-04-12]
タイトル: [例: Supabaseでデータベースを作ってみた]
絵文字: [例: 🗄️]
カテゴリ: [例: Supabase]
タグ: [例: Supabase, データベース, SQL]

今週やったこと（自分の言葉で箇条書きOK）:
・〇〇をした
・〇〇でハマった

難しかった度（1〜5）: [例: 4]
使ったツール: [例: Claude, Supabase, VSCode]

感想・振り返り:
〇〇が難しかったが、△△を理解できた。来週は〇〇をやりたい。

---

「正しい解説」パートは、上記のやったことをもとに、
初心者でもわかるよう正確な技術解説をClaudeが生成してください。
解説には以下を含めてください：
- 概念の説明（たとえを使ってわかりやすく）
- キーワードの定義（4〜5個の箇条書き）

出力は index.html のみ、weeklyDataオブジェクトの構造を維持してください。
```

---

## エントリーのデータ構造（参考）

```javascript
{
  week: 2,                          // 週番号（整数）
  date: "2026-04-12",               // 日付（YYYY-MM-DD形式）
  title: "Supabaseでデータベースを作ってみた",
  emoji: "🗄️",                      // 内容を表す絵文字ひとつ
  category: "Supabase",             // メインカテゴリ
  tags: ["Supabase", "DB", "SQL"],  // タグ（3〜5個）
  summary: "...",                   // 一行要約（Claudeが生成）
  what_i_did: "...",                // 今週やったこと（詳細）
  explanation: {
    title: "...",                   // 解説タイトル
    content: "...",                 // 本文（\nで改行）
    points: [                       // キーポイント4〜5個
      "キーワード＝説明",
      ...
    ]
  },
  reflection: "...",                // 振り返り・感想
  difficulty: 3,                    // 難易度 1〜5
  tools_used: ["Claude", "..."]     // 使ったツール
}
```

## ホスティング先が決まったら

### Netlifyの場合
1. GitHubに `weekly-ai-note` リポジトリを作成
2. このフォルダをpush
3. Netlifyで「New site from Git」→ リポジトリを選択
4. Build settingsはそのまま（静的サイト）
5. デプロイ完了 🎉

毎週更新後は `git add . && git commit -m "Week X 追加" && git push` するだけで自動デプロイ。

### GitHub Pagesの場合
1. GitHubに `weekly-ai-note` リポジトリを作成（publicで）
2. Settings → Pages → Source: main branch / root
3. URL: `https://[username].github.io/weekly-ai-note/`
